Recuperar contraseña de usuario de Windows

Esta metodología se basa en sustituir el archivo de Teclas Adhesivas (sethc.exe) por la Terminal de Comandos (cmd.exe) para lograr ejecutar comandos antes del inicio de sesión. El procedimiento inicia accediendo a las Opciones Avanzadas de la ventana de Recuperación de Windows para abrir la terminal. Desde ahí, se usa el Bloc de Notas para navegar a la ruta C:\Windows\System32. En esta ubicación, se localiza el archivo sethc.exe y se le cambia el nombre a sethc.bak (copia de seguridad). Inmediatamente, se localiza cmd.exe, se copia y la copia se renombra como sethc.exe.

Al salir y reiniciar el sistema, el usuario debe pulsar la tecla Shift cinco veces en la pantalla de inicio de sesión. Esto abrirá la terminal (gracias a la sustitución). Finalmente, se utiliza el comando net user [NombreDeUsuario] [NuevaContraseña] para cambiar la contraseña del usuario sin necesidad de ingresar al sistema.

Instalar ubuntu server y windows en la misma máquina virtual

  1. Configuración de la MV

    2. Crea una sola MV con las siguientes especificaciones: RAM: 4 GB, CPU: 2 procesadores y un Disco Duro de 40 GB (VDI).

  2. Proceso de Instalación

    3. Se recomienda instalar Windows primero porque su gestor de arranque es menos flexible que el de Linux:

    1. Instalar Windows: Arranca la MV con la ISO de Windows. Durante el particionamiento, debes reducir la partición de Windows para dejar al menos 15-20 GB de espacio SIN ASIGNAR para Ubuntu Server. Finaliza la instalación de Windows.
    2. Instalar Ubuntu Server: Carga la ISO de Ubuntu. En el paso de particionamiento, elige la opción "Manual".
    3. Particionar Linux: Utiliza el espacio sin asignar para crear las particiones de Linux: una partición Raíz (/) (formato Ext4) y una partición swap (intercambio).
    4. Gestor de Arranque: El instalador de Ubuntu detectará Windows y configurará automáticamente el Gestor de Arranque GRUB.

    Al finalizar, cuando inicies la MV, el menú GRUB aparecerá para que elijas arrancar Windows o Ubuntu Server.

Automatización de recuperación de escritorio en Windows

Al trabajar con configuraciones de monitores múltiples (pantalla extendida), Windows suele desplazar erróneamente los iconos del escritorio al monitor secundario. Esto se debe a conflictos en el Hot Plug Detect (HPD) y a la corrupción de las coordenadas guardadas en la caché de iconos del sistema.

El sistema operativo gestiona las imágenes y posiciones de los iconos mediante una base de datos local llamada IconCache.db. Cuando hay cambios bruscos de resolución o identificación de hardware (EDID), este archivo deja de sincronizarse correctamente con la pantalla principal configurada.

He desarrollado un script de automatización en Batch que realiza un mantenimiento en caliente del entorno gráfico:

  1. Finalización selectiva: Detiene el proceso explorer.exe para liberar el bloqueo del archivo.
  2. Limpieza de persistencia: Elimina la base de datos de iconos en %userprofile%\AppData\Local\.
  3. Restauración del entorno: Reinicia el Shell de Windows, forzando la regeneración de una caché limpia y el re-posicionado de iconos según la jerarquía de monitores actual.

@echo off taskkill /f /im explorer.exe del /a /f /q "%userprofile%\AppData\Local\IconCache.db" start explorer.exe

El resultado es que dispongo de una solución de soporte técnico nivel 1 que reduce el tiempo de corrección manual de minutos a segundos, garantizando la integridad visual del entorno de trabajo del usuario.
